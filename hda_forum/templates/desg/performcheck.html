Perform Check Process

Perform Check

Perform check task for to validate the design for given particular SAM boundary and verify 200+ validation checks using MapInfo validator tool and also validate the SAM, NAT and ADA boundary in PNI with reference of T18.

Boundary Check

Boundary check is the first step of Pc task, this check we are going to validate PNI boundary and T18 boundary. for ideal case both should be match each other. if you find any major boundary issue then we should raise flag to AP Team. AP team will look at the mismatch and align the boundary in PNI if required. boundary check will be done by two ways.

  • PNI Boundary Check
  • T18 & PNI boundary comparison

PNI Boundary Check

This check we have to verify SAM, NAT and ADA boundary, these boundarys should not be overlap with any other SAM boundary. SAM boundary should be lie with in FSA. if you find any of above issue we should send mail with PNI Boundary snap to AP team. they will look into PNI Boundary and correct accordingly in PNI.

T18 & PNI boundary comparison

T18 Vs PNI Boundary comparion will be done using tool (MapInfo Tool). to run this tool we need few inputs 1. T18 Files 2. PNI Boundary SAM,NAT and ADA layer. this file we can get it from PNI, need to export from PNI. ones we have these two files then run Mapinfo tool in DESG drop down Compare boundary. ones tool run is completed then you will get mismatch report in output folder, also get mail template with mismatched boundary snaps. finally verify the template mail and send to AP team. if PNI and T18 boundarys match exactly each other then u will get popup message saying no boundary mismatch then its fine go ahead further setps.

T18 Trimming

Trimming is the process of removing data from T18, which is irelavent to the that particular SAM boundary. this process also need to be done by tool. this tool retain the network entities which all are present in given boundaay, can be SAM or NAT boundary. if HFC only SAMs we should use SAM boundary for trimming, if MTM and Patial ONSA SAMs we should use NAT boundary for trimming.

T18 Validation and Fix

T18 Validation and fix will be done by MapInfo tool. there are 200+ checks all checks will be checked using this tool. as part of this validation process node split can be done based on node split defined in NDD Document,which is available in Sharepoint location. ones validation and fix is completed then we can get checklist results csv file. this file contains all checks related data.

MTM Document Check

MTM Document check required for FTTx SAMs, there are few set of document required for process FTTN SAMs.

Document Required:

  1. SLD
  2. Splicing Document
  3. CCU Workbook
  4. Redline Document
  5. BOMBOQ Minima

Boundary Demotion

Boundary demotion is the process of removing Teltra support structure, copper network, fibre network and HFC network design which already present in PNI.

Demotion Steps

  • Create Job in PNI
  • Export Design from PNI
  • Export OSP Report from PNI
  • Telstra support structure demotion
  • Fibre Network Demotion (if required)
  • Copper network demotion (if required)
  • HFC network demotion (if required)
  • Post the job

MPS Boundary Promotion

MPS promotion is the process of promoting multi premises sites boundarys in PNI. MPS boundarys containing MDUs & SDU (Multi duelling unit & single duelling Unit)

MPS Boundary extraction from T18

Boundary table contains all the MPS boundary. this MPS extraction process only bring MPS boundary from boundary table with specific table format (importable) with the technology type using Mapinfo tool. ones MPS boundary extraction is completed then import MPS boundary layer in PNI. if error is zero then we can go ahead further, if you encounter any issues like boundary overlap or boundary avoidences error then we have to fix the issue adjusting MPS boundary in map object. before MPS boundary promotion make sure Technology type should not be blank.

Loc ID verification in T18 Vs PNI

This check will be start ones we get address files from LDM. this check mainly for validate all the MPS boundary should be served by GTD, and GTD, PCD(SL) should be inside MPS Boundary, if not we need to verify the sl location in state and Google Map exact location of SL then move the PCD accordingly in PNI.

PNI Validation Run

PNI Validation run is the process of Pre-Validating GML file in PNI. this is dummy run in PNI to validate all the network entities have meet the pre-validation. if anything failed during validation run then failed network entities will be captured in GML Summary report. SpatialNET application only accept GML file to validate the design in PNI. in order to convert T18 file to GML we need GDSS portal, which will take T18 file as input and give few output files like GML, Location trace report and etc..
